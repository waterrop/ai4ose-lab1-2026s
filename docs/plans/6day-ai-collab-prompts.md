# AI协作提示词包（6天通用）

以下提示词按“概念 -> 实现 -> 调试 -> 输出改进”四类组织，可直接复制到 Cursor/Chat 模型中使用。

## 1) 概念教练提示词

### Prompt A：知识点讲透
请用“现象 -> 机制 -> 代码位置 -> 最小例子 -> 常见误解”的结构解释这个知识点：`<知识点>`。
约束：
1. 结合 rCore ch<章节号> 的上下文；
2. 用 1 张 ASCII 流程图表示关键路径；
3. 给 2 个学生常见误区和纠正方式。

### Prompt B：对比学习
请对比 `<机制A>` 和 `<机制B>`：
1. 相同点；
2. 差异点；
3. 各自在 ch<章节号> 中的必要性；
4. 错配会导致什么 bug。

## 2) 实现教练提示词

### Prompt C：最小可通过方案
基于 `ch<章节>/exercise.md` 的要求，给我“最小可通过测例”的实现路线图。
要求：
1. 按文件、函数、改动点列出；
2. 每步说明输入/输出与预期日志；
3. 不一次性给全量代码，只给当前步需要的核心片段。

### Prompt D：分层实现
把本练习拆成 3 层：
L1 通过测例、L2 提升健壮性、L3 提高可维护性。
对每层分别给出：
1. 需要修改的函数；
2. 验证命令；
3. 完成标志。

## 3) 调试教练提示词

### Prompt E：日志驱动排查
这是当前报错与日志：
`<粘贴日志>`

请给出：
1. 最可能 3 个根因（按概率排序）；
2. 每个根因的验证步骤（最多 3 步）；
3. 最短修复路径；
4. 修复后如何防止回归。

### Prompt F：测试不通过定位
`./test.sh exercise` 在 `<case_name>` 失败。
请输出：
1. 失败语义对应的内核职责；
2. 需要重点排查的数据结构；
3. 最快插桩点（函数 + 打印字段）；
4. 常见错误分支清单。

## 4) 费曼输出教练提示词

### Prompt G：讲稿审阅
这是我的讲解稿：
`<粘贴讲稿>`

请严格审查并输出：
1. 逻辑跳步位置；
2. 术语误用；
3. 缺失前提；
4. 面向大二学生的改写版（5分钟）。

### Prompt H：反向提问
你扮演助教，针对 `<章节主题>` 向我连续提 8 个问题（由浅入深）。
要求：
1. 至少 2 个“为什么不是另一种方案”的问题；
2. 至少 2 个“定位 bug”问题；
3. 每题附简短评分标准。

## 5) 教程改进提示词

### Prompt I：学生自学版改写
请把 `ch<章节>` 教程改写成“学生自学版”。
结构固定为：
1. 学习目标
2. 先修知识
3. 最小实践任务
4. 关键代码导航
5. 常错点
6. 验收清单

### Prompt J：个性化增强
学习者画像：`Rust中等、OS基础一般、实践时间有限、容易在调试中迷失`。
请对 `ch<章节>` 提供：
1. 低挫败学习路径；
2. 90分钟卡关应急策略；
3. 最小复盘模板。
